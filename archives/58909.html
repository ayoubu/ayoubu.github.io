scrapy 的基础概念

在之前学的requests + selenium 已经可以解决90%的爬虫需求了，而学习 scrapy 是为了让我们爬虫更快，更强。从而提高我们爬虫的效率和速度。

什么是 scrapy

Scrapy 是一个为了爬取网站数据 ，提取结构性数据 而编写的应用框架 ，我们只需要实现少量的代码，就能够快速的抓取。（框可能包含了多个模块，而模块只是负责单一功能）

Srcapy 使用了 Twisted[‘twistid’] 异步网络框架，可以加快我们的下载速度。

Scrapy 中文文档

  • 异步和非阻塞的区别

异步：调用在发出之后，这个调用就直接返回，不管有无结果

非阻塞：关注的是程序在等待调用结果（消息，返回值）时的状态，指在不能立刻得到结果之前，调用不会阻塞当前线程。

scrapy的爬虫流程：

scrapy 模块说明：

模块名 说明 是否需要实现
Scrapy Engine (引擎) 总指挥：负责数据和信号在不同模块间的传递 scrapy 已经实现
Scheduler (调度器) 一个队列，存放引擎发过来的 requests 请求 scrapy 已经实现
Downloader (下载器) 下载引擎发来的requests请求，并返回给引擎。 scrapy 已经实现
Spider (爬虫) 处理引擎发过来的response，提取数据，提取url，并交给引擎 需要手写
Item Pipeline (管道) 处理引擎传过来的数据，比如存储 需要手写
Downloader Middlewares (下载中间件) 可以自定义的下载扩展，比如设置代理 一般不用手写
Spider MiddlewareSpider (爬虫中间件) 可以自定义 requests 请求和进行 response 过滤 一般不用手写

scrapy 的数据流程

  • 调取器 —> request 对象 —> 引擎 —> 下载中间件 —> 下载器
  • 下载发送请求，获取响应 —> response —> 下载中间件 —> 引擎 —> 爬虫中间件 —> spider
  • spider 提取数据 —> 引擎 —> pipeline
  • spider 提取的 url 地址 —> 构造 request 对象 —> 爬虫中间件 —> 引擎 —> 调度器

spider 的数据传到 pipeline

在我们编写完 spider 中的 parse 函数后，想要将我们提取到的数据传给管道，此时只需要使用：yield item 方法就好了。

为什么要使用yield？

让整个函数变成一个生成器，变成 generator (生成器) 的好处就是每次遍历的时候挨个读到内存中，不会导致内存的占用量瞬间变高。python3 中 range 和 python2 中的 xrange 同理。

从pipeline 的字典形式可以看出来，pipeline 可以有多个，而且确实可以定义多个。

为什么需要多个 pipeline：

  1. 可能会有多个 spider，不同的 pipeline 处理不同的 item 的内容
  2. 一个 spider 的内容可能需要不同的操作，比如存入不同的数据库中
  3. 判断数据的来源可以通过给item中添加键来完成，也可以通过判断spider的名称来判断：if spider.name == 'itcast'，因为我们在定义爬虫的时候有定义爬虫的名字。

注意：

  1. pipeline 的权重越小优先级越高
  2. pipeline 中 process_Item 方法名不能改为其它名称

简单设置 log

为了让输出端的，我们希望看到的内容更清晰一些，我们可以在 setting.py 中设置显示的 log 级别，通过添加一行代码（全部大写）：LOG_LEVEL = WARNING，默认显示的是 debug 级别的 log 信息。

在 Scrapy 中我们可以通过 print 函数打印过程信息，但是这种方式的缺点是，当创建的爬虫多了以后，不清楚到底是哪个爬虫打印的，同时打印的信息也无法保存在本地中。此时我们可以使用 python 中的 logging 模块，将我们想要打印的信息，以log的方式打印出来。我们还可以在setting 中设置，将信息保存下来。

在我们需要输出信息的地方导入logging模块

logging 模块使用总结：

  • srcapy 中使用
    • setting 中设置 LOG_LEVEL=”WARNING”
    • setting 中设置 LOG_FILE=”./a.log” # 设置日志保存的位置，设置后终端不会显示日志的内容，但是会保存在本地
    • import logging，实例化 logger 的方式在任何文件中使用logger 输出内容
  • 普通项目中使用
    • import logging
    • 设置日志输出的样式，格式：logging.basicConfig(...)
    • 实例化一个logger：logger=logging.getLogger(__name__)
    • 在任何py文件中调用 logger 即可

实现翻页请求

在我们使用requests 模块的时候，我们的翻页请求是通过找到下一页的地址，然后直接调用 requests.get(url) 来实现的。

而在 scrapy 中翻页的原理一样，只不过我们需要将下一页的 url 地址组合成一个 requests 对象，来实现翻页的请求的。

关于 scrapy.Request 知识点：

在文档中方括号的参数表示可选参数，比较重要的为：

  • callback：指定传入的 url 交给哪个解析函数去处理
  • meta：实现在不同的解析函数中传递数据，meta 默认会携带部分信息，比如下载延迟，请求深度 等。
  • dont_filter：让 scrapy 的去重不会过滤当前 url，scrapy 默认有url去重的功能。此参数对于需要重复请求的url很重要。

Scrapy shell

Scrapy shell 是一个交互终端，我们可以在未启动spider的情况下尝试及调试代码，也可以用来测试 XPath 表达式。

使用方法：scrapy shell 网址

常用方法：

  • response.url：当前响应的url地址
  • response.request.url：当前响应对应的请求 url 地址
  • response.headers：响应头
  • response.body：响应体，也就是 html 代码，默认是 byte 类型
  • response.requests.headers：当前响应的请求头
