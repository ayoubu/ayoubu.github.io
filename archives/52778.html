MongoDB简介

MongoDB简介

MongoDB是一个基于分布式文件存储的数据库，由C++语言编写，旨在为WEB应用提供可扩展的高性能数据存储解决方案。

MongoDB将数据存储为一个文档，数据结构由键值对(Key : value)组成，MongoDB文档类似于JSON对象，字段值可以包含其它文档，数组及文档数组。

MongoDB服务端可运行在Linux、Windows和Mac OS平台，支持32位和64位应用，默认端口为27017。本文所展示的为Windows平台的安装及使用。

MongoDB支持各种编程语言：Python、Java、C++、PHP、C#等多种语言。

MongoDB的优势

  • 易扩展： NoSQL（非关系型数据库）数据库种类繁多，但是一个共同的特点都是去掉关系型数据库的关系特性。数据之间无关系，这样就非常容易扩展。
  • 大数据量，高性能： NoSQL数据库都具有非常高的读写性能，尤其在大数据量下，同样表现优秀。得益于它的无关系性，数据的结构简单
  • 灵活的数据模型： NoSQL无需事先为要存储的数据建立字段，随时可以存储自定义的数据格式。而在关系型数据库里，增删字段是一件非常麻烦的事情。

缺点： 由于它每条数据都是单独存储的，因此在存储过程中可能会包含大量重复信息，非常占用存储空间。

MongDB的安装（windows）

  1. 首先下载MongDB的社区版，我这边下载的是64位的版本（下载）。
  2. 打开.msi的安装文件，按照指示一路点击Next就好，如果想更改安装目录，需要选择Custom。在最后我选的是以Web用户的方式来使用MongDB。
  3. 在最后不要勾选Install MongoDB Compass。MongoDB Compass 是一个图形界面管理工具，这里不安装也没有问题，可以自己安装一个图形界面管理工具，比如Robo3T。

配置MongoDB

MongoDB的安装步骤比较简单，但是配置较为繁琐。

首先在MongoDB的data文件夹里新建一个db文件夹和一个log文件夹，然后在log文件夹下新建一个mongo.log文件。

将安装路径(..\MongoDB\bin)添加到环境变量path中，此时打卡cmd窗口，运行mongo命令，此时得到下图结果。

提示我们连接失败，是因为我们还没有启动MongoDB服务。在cmd中通过以下命令启动：mongod -- dbpath "..\MongoDB\data\db“。（注：后面这个data的文件夹也可以创建在别的路径，但是data文件夹中得包含log文件夹和db文件夹，如果没有提前创建db文件夹，则服务器会启动失败。）

打开浏览器，输入127.0.0.1:27017，尝试连接数据库，得到下图所示结果，说明启动成功。

配置MongoDB的自启动：

首先关闭mongoDB的服务，在cmd窗口可以按Ctrl + C来停止。然后以管理员身份运行cmd窗口，输入命令：mongod --logpath "..\MongoDB\data\log\mongo.log" --logappend --dbpath "..\MongoDB\data\db" --directoryperdb --serviceName "MongoDB" --install ，如果没有报错，则说明添加成功，可以使用win+R，然后输入services.msc命令进行查看。

我的配置命令为：mongod --logpath "D:\Data\MongoDB\data\log\mongo.log" --logappend --dbpath "D:\Data\MongoDB\data\db" --directoryperdb --serviceName "MongoDB" --install只需要替换里面的文件夹路径即可。

这里显示的MongoDB Server是自动启动的，我们把它设置为手动的。然后再cmd窗口中输入：net start mongodb。

问题：

  1. 我在启动的时候出现了拒绝访问，通过管理员权限运行cmd窗口后可以正常启动。
  2. 如果出现服务没有相应控制功能，1)运行sc delete mongodb删除服务，再运行一次配置服务命令。

Windows MongoDB服务自动关闭

我在使用windows 系统的MongoDB服务时，服务总是自动中断，通过下述方式解决。

  1. 在命令行中输入：mongod --remove
  2. 删除之前配置的db文件夹中的mongod.lock和storage.bson两个文件。
  3. 重新配置自动启动mongod --logpath "D:\Data\MongoDB\data\log\mongo.log" --logappend --dbpath "D:\Data\MongoDB\data\db" --directoryperdb --serviceName "MongoDB" --install
  4. 启动服务：net start mongodb

注意，以上操作，会清楚数据库的信息。

关于Database的基础命令

在安装成功后，启用MongoDB的服务，然后在命令行窗口输入：mongo，就可以进入mongoDB的命令行模式，然后就可以执行一些操作。

  1. 查看当前的数据库：db
  2. 查看所有的数据库：show dbs / show databases
  3. 切换数据库：use db_name
  4. 删除当前的数据库：db.dropDatabase()
  5. 删除数据集合：db.集合命.drop()

MongoDB数据库操作

mongoDB插入数据

  • db.collection.insert({}) 插入数据，_id存在就会报错
  • db.collection.save({}) 插入数据，_id存在就会更新

mongoDB更新操作

  • db.collection.update({name:"大宝"},{name:"大大宝"})

此操作会直接用{name:”大大宝”}这条数据替换掉包含{name:”大宝”}的这条数据。

  • db.collection.update({name:"大宝"},{$set:{name:"大大宝"}})

此操作只会更新包含{name:”大宝”}的这条数据中的”name”这个字段，其它信息不变。

  • db.collection.update({name:"大宝"},{$set:{name:"大大宝"}},{multi:true})

第三个参数设置为{multi:true}，表示会更新所有符合要求的数据，不然只会更新查找到的第一条数据。（注：第一种替换的方式时不能用这个参数，因为它这可以和$连用。）

mongoDB删除数据

  • db.collection.remove({name:"大大宝},{justOne:true})

删除第一条出现的包含{name:”大大宝}这个信息的数据，第二个参数可以不写，不写默认删全部。

数据查询

  • find() 方法：db.collection.find({条件文档}) — 查询全部
  • findOne() 方法：db.collection.findOne(条件文档) — 查询，返回第一个
  • pretty() 方法：db.collection.find({条件文档}).pretty() — 将结果格式化

自定义查询

使用$where后面写一个函数，返回满足条件的数据，这种写法(js语句)一般是在终端中写时才会用到。

比如查询年龄大于30的学生：

投影

选择返回结果的字段。在查询到的返回结果中，只选择必要的字段。表达式：db.collection.find({},{字段名称:[0,1]})，除了id外其它字段只能全部0或者1，写的为0时，没写的默认为1。_id 默认会显示，想不显示得设置为1。

统计个数

count()方法用于统计查询结果中文档的条数。

表达式：

  • db.collection.find({条件}).count() 此时count()中写条件是无效的
  • db.collection.count({条件})

消除重复

distinct()方法用于对数据进行去重操作。表达式：db.collection.distinct('去重字段', {条件})，如：db.collecttion.distinct('hometown',{age:{$gt:18}})。

distinct() 方法返回的是一个该字段中不重复值的数据列表。

mongoDB表达式中的运算符

比较运算符

  • 等于：默认是等于判断，没有运算符
  • 小于：$lt —> (less than)
  • 小于等于：$lte —> (less than equal)
  • 大于：$gt —> (greater than)
  • 大于等于：$gte —> (greater than equal)
  • 不等于：$ne —> (no equal)

eg：db.collection.find({age:{$gte:18}})

范围运算符

使用"$in"，"nin"判断是否在某个范围内，比如查询年龄为18、28、38的学生。db.collection.find({age:{$in:[18,28,38]}}) 。

逻辑运算符

  • and: 在json中写多个条件即可。比如查询年龄大于或等于18，并且性别为男的学生：db.collection({age:{$gte:18},gender:"男"})。
  • or: 使用$or，值为数组，数组中每个元素为json。比如查询年龄大于18，或者性别为男的学生：db.collection.find({$or:[{age:{&gte:18}},{gender:"男"}]})。

查询年龄大于18或性别为男生，并且姓名是郭靖：db.collection.find({$or:[{age:{$gte:18}},{gender:"男"}], name:"郭靖"})

mongoDB中的正则表达式

使用//或者$regex编写正则表达式。

比如:

db.collection.find({sku:/^abc/}) — 表示以abc开头的。

db.collection.find({sku:{$regex:'789$'}}) — 表示以789结尾的。

mongoDB中的一些附加方法

  • limit() 方法：用于读取指定数量的文档。表达式：db.collection.find().limit(NUMBER)，比如查询两条：db.collection.find().limit(2)。
  • skip() 方法：用于跳过指定数量的文档。 表达式：db.collection.find().skip(NUMBER)，比如跳过两条数据：db.collection.find().skip(2)。
  • 同时使用，两者也可以同时使用，如：db.collection.find().limit(4).skip(5) 或者 db.collection.find().skip().limit()。

数据的备份和恢复

备份语法：

mongodump -h dbhost -d dbname -o dbdirectory

  • -h: 服务器地址，也可以指定端口号
  • -d: 需要备份的数据库名称
  • -o: 备份的数据存放的位置，此目录中存放着备份出来的数据

mongodump -h 192.168.196.128:27017 -d test -o ~/Desktop/test1bak

恢复语法：

mongorestore -h dbhost -d dbname --dir dbdirectory

  • -h：服务器地址
  • -d：需要恢复的数据库实例
  • –dir：备份数据所在的位置

mongorestore -h 127.0.0.1:27017 -d test --dir ~/Desktop/test1bak/test1
