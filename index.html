scrapy 的基础概念

在之前学的requests + selenium 已经可以解决90%的爬虫需求了，而学习 scrapy 是为了让我们爬虫更快，更强。从而提高我们爬虫的效率和速度。

什么是 scrapy

Scrapy 是一个为了爬取网站数据 ，提取结构性数据 而编写的应用框架 ，我们只需要实现少量的代码，就能够快速的抓取。（框可能包含了多个模块，而模块只是负责单一功能）

Srcapy 使用了 Twisted[‘twistid’] 异步网络框架，可以加快我们的下载速度。

Scrapy 中文文档

  • 异步和非阻塞的区别

异步：调用在发出之后，这个调用就直接返回，不管有无结果

非阻塞：关注的是程序在等待调用结果（消息，返回值）时的状态，指在不能立刻得到结果之前，调用不会阻塞当前线程。

scrapy 模块说明：

模块名 说明 是否需要实现
Scrapy Engine (引擎) 总指挥：负责数据和信号在不同模块间的传递 scrapy 已经实现
Scheduler (调度器) 一个队列，存放引擎发过来的 requests 请求 scrapy 已经实现
Downloader (下载器) 下载引擎发来的requests请求，并返回给引擎。 scrapy 已经实现
Spider (爬虫) 处理引擎发过来的response，提取数据，提取url，并交给引擎 需要手写
Item Pipeline (管道) 处理引擎传过来的数据，比如存储 需要手写
Downloader Middlewares (下载中间件) 可以自定义的下载扩展，比如设置代理 一般不用手写
Spider MiddlewareSpider (爬虫中间件) 可以自定义 requests 请求和进行 response 过滤 一般不用手写

scrapy 的数据流程

  • 调取器 —> request 对象 —> 引擎 —> 下载中间件 —> 下载器
  • 下载发送请求，获取响应 —> response —> 下载中间件 —> 引擎 —> 爬虫中间件 —> spider
  • spider 提取数据 —> 引擎 —> pipeline
  • spider 提取的 url 地址 —> 构造 request 对象 —> 爬虫中间件 —> 引擎 —> 调度器

让整个函数变成一个生成器，变成 generator (生成器) 的好处就是每次遍历的时候挨个读到内存中，不会导致内存的占用量瞬间变高。python3 中 range 和 python2 中的 xrange 同理。

而在 scrapy 中翻页的原理一样，只不过我们需要将下一页的 url 地址组合成一个 requests 对象，来实现翻页的请求的。

关于 scrapy.Request 知识点：

在文档中方括号的参数表示可选参数，比较重要的为：

  • callback：指定传入的 url 交给哪个解析函数去处理
  • meta：实现在不同的解析函数中传递数据，meta 默认会携带部分信息，比如下载延迟，请求深度 等。
  • dont_filter：让 scrapy 的去重不会过滤当前 url，scrapy 默认有url去重的功能。此参数对于需要重复请求的url很重要。

页面等待

  • 为什么需要等待？

    • 如果网站采用了动态html技术，那么页面上的部分元素出现时间便不能确定，这个时候就可以设置一个等待时间，强制要求在时间内出现，否则报错

  • 强制等待：time.sleep(10)-强制等待10秒

selenium 使用的注意点

  1. 获取文本和属性
    • 先定位到元素，然后调用.text() 或者 get_attribute 方法来取
  2. selenium 获取的页面是浏览器中 elements 的内容
  3. find_element 和 find_elements 的区别
    • find_element 返回的是一个 element，如果没有会报错
    • find_elements返回一个列表，没有就是空列表
    • 在判断是否有下一页的时候，使用find_elements来根据结果列表长度判断
  4. 如果页面中含有iframe、frame，需要先调用 driver.switch_to.frame的方法切换到frame中才能定位元素
  5. selenium 请求第一页的时候会等待页面加载完了之后再获取数据，但是在点击翻页之后，会直接获取数据，此时可能会报错，因为数据还没有加载出来，需要time.sleep(3) 来强制等待。
  6. selenium 中 find_element_by_class_name 只能接收一个class对应的一个值，不能传入多个。

验证码的识别

  • url 不变，验证码不变

    • 请求验证码的地址，获得响应，调用接口进行识别

  • url 不变，验证码会变

    • 思路：对方服务器返回验证码的时候，会将每个用户的信息和验证码进行一个对应，之后，在用户发送post请求的时候，会对比post请求中发的验证码和当前用户真正的存储在服务器端的验证码是否相同
    1. 实例化 session
    2. 使用 session 请求登录页面，获取验证码的地址
    3. 使用 session 请求验证码，识别
    4. 使用 session 发送 post 请求

  • 使用 selenium 登录，遇到验证码

    • url不变，验证码不变，同上

    • url不变，验证码会变

      1. selenium 请求登录页面，同时拿到验证码的地址
      2. 获取登录页面中的 driver 中的 cookie，交给 requests 模块发送验证码的请求，识别
      3. 输入验证码，点击登录
      • 实验不行就使用截图工具对页面验证码区域进行自动截图，此时需要固定浏览器的宽高，不然截图的区域会变。

正则表达式

正则表达式的定义：就是用事先定义好的一些特定字符、及这些特定字符的组合，组成一个“规则字符串”，这个“规则字符串” 用来表达对字符串的一种过滤 逻辑。

常用正则表达式的方法：

  • re.compile() — 编译
  • pattern.match() — 从头找一个
  • pattern.search() — 找一个
  • pattern.findall() — 找所有（找出所有）
  • pattern.sub() — 替换

python中原始字符串r的用法

在字符串前加r，表示这个字符串中的所有都是字符格式，此时一些特殊字符也会全部被当作普通字符来处理。

注： 在python正则表达中尽可能的使用原始字符串，待匹配的字符串中看到什么就在正则表达式写什么，就不会有问题。

re.compile的使用

re.compile()是re模块中的一个方法，它的第一个参数是要编译的参数，第二个参数是模式（比如可以设置为：re.DOTALL/re.S，表示匹配一些特殊字符如\n等），返回值是re.Pattern的一个对象。有了这个对象以后就不用在正则匹配中去添加正则表达式了，直接使用这个对象就会默认使用创建对象时的正则表达式。这样做的好处就是正则表达式只会生成一次，在我们多次使用正则匹配的时候可以帮我们节省运行时间。

贪婪匹配和非贪婪匹配

在匹配时使用*或者+号就是属于贪婪匹配。贪婪匹配就是尽可能多的向后匹配。

此时匹配到了两条数据，是因为a中有换行符，虽然不能直接看到，但是通过运行a就可以看到。而.运算符是不能匹配到换行符的。

贪婪匹配：

为了演示贪婪匹配和非贪婪匹配，我们可以在findall()方法中，添加参数，使得.运算符可以匹配到换行符，通过设置模式为：re.S或者re.DOTALL，此时我们得到的结果只有一条，如下所示：

非贪婪匹配：

非贪婪匹配就是尽可能少的匹配，发现一个满足条件的就返回，然后接着找。非贪婪匹配就是在刚才贪婪匹配后面加个?号就行。此时得到的就是两条数据。

加不加（）的区别

不加()括号返回的是匹配的是整体，级整个字符串的内容。加()只会返回括号里面匹配到的内容。

正在表达式使用的注意点

  • re,findall("a(.*?)b","str")，能够返回括号中的内容，括号前后的内容起到的是过滤的作用。
  • 原始字符串r，待匹配字符串中有反斜杠的时候，使用r能够忽视反斜杠带来的转义的效果。
  • 点号默认情况匹配不到\n
  • \s能够匹配空白字符，不仅仅包含空格，还有\t,\r,\n。

/bookstore/book[last()-1] 选取属于bookstore子元素的倒数第二个book元素。
/bookstore/book[position()<3] 选取最前面两个属于bookstore元素的子元素的book元素

lxml库

  • 使用入门：（需要安装 lxml 第三方模块）
    • 导入 lxml 的 etree 库：from lxml import etree
    • 利用 etree.HTML，将字符串转化为Element对象
    • Element对象具有xpath的方法：html = etree.HTML(text)
  • lxml 可以自动修正 html 代码
  • 查看转换后的 html 文本，可以使用:etree.tostring(html).decode()

XPath学习重点

  • 使用xpath helper或者是chrome中的copy xpath都是从elements中提取的数据，但是爬虫获取的是url对应的响应，往往和elemens中的内容不一样。
  • 获取文本的方法
    • a/text() —> 获取a标签下的文本
    • a//text() —> 获取a标签下的所有文本
    • //a[text()='下一页'] —> 选择文本为下一页三个字的a标签
  • @符号的使用方法
    • a/@href —> 获取a标签下的所有网址
    • //ul[@id="detail-list"] —> 选取id为”detail-list”的ul标签
  • //的使用
    • 在XPath开始的时候表示从当前html中任意位置开始选择
    • li//a表示的是li下任何一个a标签
  • xpath 的包含
    • //div[contains(@class,'i')] —> 表示div中class包含i的标签

lxml 使用注意点

  • lxml 能够修正 HTML 代码，但是可能会改错了
    • 使用 etree.tostring 观察修改之后的 html 的样子，根据修改之后的 html 字符串写 xpath
  • lxml 能够接收 bytes 和 str 的字符串，如果一个转换错误，就试试另一个。
  • 提取页面数据的思路
    • 先分组，取到一个包含分组标签的列表
    • 再遍历，取其中每一组进行数据的提取，不会造成数据的对应错乱

实现爬虫的套路

准备url：

  • 准备start_url
    • url地址规律不明显，总数不确定
    • 通过代码提取下一页的url
      • xpath
      • 寻找url地址，部分参数在当前的响应中（比如，当前页码数和总页码数在当前响应中）
  • 准备url_list
    • 页码总数明确
    • url地址规律明显

发送请求，获取响应：

  • 添加随机的User-Agent，反反爬虫
  • 添加随机的代理IP，反反爬虫
  • 在对方判断出我们是爬虫之后，应该添加更多的headers字段，包括cookie
  • cookie的处理可以使用session来解决
  • 准备一堆能用的cookie，组成cookie池
    • 如果不登录
      • 准备开始能够请求对方网站的cookie，就是接收对方网站设置在response的cookie
      • 下一次请求的时候，使用之前的列表中的cookie来请求
    • 如果登录
      • 准备多个账号
      • 使用程序获取每个账号的cookie
      • 之后再随机的选择cookie，请求登录之后才能访问的网站

提取数据：

  • 确定数据的位置
    • 如果数据在当前的url地址中
      • 提取的是列表页的数据
        • 直接请求列表页的url地址，不用进入详情页
      • 提取的是详情页的数据
        1. 确定url
        2. 发送请求
        3. 提取数据
        4. 返回
    • 如果数据不在当前的url地址中
      • 在其它的响应中，寻找数据的位置
        1. 从network中从上往下找
        2. 使用chrome中的过滤条件，选择除了js，css，img之外的按钮
        3. 使用chrome的search all file，搜索数字和英文
  • 数据提取
    • xpath，从html中提取整块的数据，先分组，之后每一组再提取
    • re，提取max_time，price，html中的json字符串
    • json，从对应的响应中直接得到数据
  • 保存数据
    • 保存在本地，text，json，csv
    • 保存在数据库

如何在命令行运行 python 程序

最简单的情况就是，在当前python程序所在的路径下打开命令行终端，然后直接输入：python 程序名.py 就可以运行。

但是有时候我们需要向程序中输入参数，我们可以直接使用x = input() ，但是这样会多一步交互，不够简洁。

在命令行中，如果我们输入：python 程序名.py 1 2 3，我们实际上可以得到一个列表：['程序名.py', '1', '2', '3']，而后面的这些输入其实是可以作为参数传给程序的。

我们只需要在程序中使用sys.argv[]来接收就行，如果我们写x = sys.argv[1]表示我们接收的值为第一个，即1，此时x=1。
